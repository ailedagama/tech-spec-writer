# tech-spec-writer

> B端招标技术规范书 AI 编写助手 — 面向核电、高新制造、半导体、教育等行业，基于客户背景调研与产品功能清单，交互式生成可直接用于招标立项的技术规范书草稿。

---

## ✨ 功能概览

| 能力 | 说明 |
|---|---|
| 🔍 客户背景自动调研 | 输入客户名称与官网，AI 自动搜索公司介绍、采购历史、合规约束等公开信息 |
| 📋 功能清单驱动生成 | 上传或粘贴产品功能清单，AI 解析后驱动框架设计与功能章节内容填充 |
| 🏗️ 行业适配框架生成 | 根据行业类型（核电/制造/半导体/教育）自动适配章节结构与合规标准 |
| 📝 逐章交互式填充 | 每章完成后等待用户确认，支持随时修改，不一次性生成全文 |
| ✅ 规范性自动校验 | 全文生成后自动检查结构完整性、条款一致性、标准引用规范等 5 个维度 |
| 🔄 修订闭环 | 支持针对校验问题进行二次修订，输出修订前后对比 |

---

## 🚀 快速开始

### 触发方式

在对话中直接说明需求即可触发，例如：

```
帮我生成台山核电的智能问数技术规范书
/tech-spec-writer 帮我写一份防城港核电的ChatBI采购技术规范书
```

### 需要准备的信息

启动后 AI 会引导您填写以下内容：

| 项目 | 是否必填 | 说明 |
|---|---|---|
| 客户名称 + 官网 | ✅ 必填 | 用于网络调研客户背景 |
| 行业类型 | ✅ 必填 | 核电 / 高新制造 / 半导体 / 教育 / 其他 |
| 需求概要 | ✅ 必填 | 采购产品、应用场景、业务目标 |
| **产品功能清单** | ⭐ 强烈建议 | 粘贴功能模块列表或上传文档，直接驱动功能章节生成 |
| 技术限制条件 | 选填 | 如私有化部署、等保要求、系统集成等 |
| 预期交付时间 | 选填 | 如 2025年Q4、6个月内 |

> 💡 **建议提供功能清单**：若提供产品功能清单，AI 将基于清单驱动第四章「功能要求」的模块划分与内容填充，生成内容与实际产品高度匹配；若未提供，AI 将基于产品类型推断，所有功能点会标注「📌 基于产品类型推断，建议核实」。

## 📋 完整交互流程

```
Step 1  客户信息采集
        └── 客户基本信息 + 行业 + 需求概要 + 产品功能清单
              ↓
Step 2  客户背景调研 & 框架生成（AI 自动执行）
        ├── 2.1 网络调研（官网、采购记录、行业动态）
        ├── 2.2 客户背景摘要输出
        ├── 2.2.1 功能清单结构化解析
        │         ├── 核心功能模块归类
        │         ├── 扩展/管理功能归类
        │         └── 行业缺口功能建议
        └── 2.3 章节框架草案输出（含字数预估）
              ↓
Step 3  用户确认框架
        └── A 确认 / B 局部修改 / C 完全自定义
              ↓
Step 4  逐章内容填充
        ├── 每章填充完展示，等待确认再继续
        ├── 内容分层标记：【通用】【行业定制】【客户定制】⚠️
        └── 详细级章节必须展开至三级条款（X.X.X）
              ↓
Step 5  规范性自动校验
        └── 结构完整性 / 条款一致性 / 标准引用 / 语气规范 / ⚠️ 标注检查
              ↓
Step 6  用户反馈 & 二次修订
        └── 逐条处理，输出修订前后对比，汇总修订结果
              ↓
        输出：技术规范书草稿（Markdown，可导入 Word / 飞书）
```

## 📐 文档规模参考

| 章节 | 深度等级 | 预估字数/章 |
|---|---|---|
| 第一章 总则 | 🔵 概要级 | 900 ~ 1,400 字 |
| 第二章 项目概述 | 🔵 概要级 | 800 ~ 1,200 字 |
| 第三章 系统技术架构要求 | 🔴 详细级 | 3,500 ~ 5,500 字 |
| 第四章 功能要求 | 🔴 详细级 | 5,500 ~ 9,000 字 |
| 第五章 接口与集成要求 | 🔴 详细级 | 2,500 ~ 3,800 字 |
| 第六章 性能与可靠性要求 | 🟠 标准级 | 1,500 ~ 2,200 字 |
| 第七章 安全与合规要求 | 🔴 详细级 | 3,000 ~ 4,500 字 |
| 第八章 交付物要求 | 🟠 标准级 | 700 ~ 1,050 字 |
| 第九章 实施与服务要求 | 🔵 概要级 | 800 ~ 1,200 字 |
| 第十章 附录 | 🔵 概要级 | 600 ~ 900 字 |
| **全文合计** | — | **约 19,800 ~ 29,900 字** |

**深度等级说明：**
- 🔴 **详细级**：必须展开至三级条款（X.X.X），关键参数以表格呈现
- 🟠 **标准级**：包含量化指标和清单，二级条款完整
- 🔵 **概要级**：定性描述为主，条款简洁

## 🔖 内容标注体系

生成内容统一使用以下标注，便于人工审核：

| 标注 | 含义 |
|---|---|
| `【通用】` | 适用于所有行业和客户的标准表述，可直接使用 |
| `【行业定制】` | 基于行业知识生成的专属内容，建议结合实际情况核实 |
| `【客户定制】` | 基于本次客户调研信息填充，信息来源已标注 |
| `⚠️ 需人工核实` | 涉及安全等级、验收数值、合规标准编号等强制参数，**必须人工确认后使用** |
| `📌 基于产品类型推断` | 未提供功能清单时的推断内容，建议对照实际产品核实 |
| `📌 建议补充` | AI 基于行业经验建议纳入但原始清单未包含的功能点 |

## ⚠️ 使用边界

**本 Skill 能做：**
- ✅ 生成技术规范书结构框架与各章节草稿内容
- ✅ 行业适配的术语规范与合规标准引用建议
- ✅ 基于产品功能清单驱动功能章节精准填充
- ✅ 自动标注需人工核实的高风险条款

**本 Skill 不做：**
- ❌ 替代人工进行最终法律与合规审核
- ❌ 自动填写核安全等级等强制性参数
- ❌ 提供商务条款、报价及合同内容
- ❌ 生成已盖章的正式招标文件

---

## 📄 输出格式

- **格式**：Markdown（标题层级对应章节编号）
- **版本标注**：`V1.0-Draft，需人工审核后正式发布`
- **附录 C**：自动汇总全文所有 `⚠️ 需人工核实` 条款清单
- **导入支持**：可直接导入飞书文档、Word、Notion 等工具
